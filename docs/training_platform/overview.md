# Обзор Платформы Экспериментов

## Цель

Нужна модульная и управляемая система для экспериментов над `NVIDIA-Nemotron-3-Nano-30B-A3B`, в которой можно:

- удобно менять конфиги обучения, reward, eval, данных и runtime;
- запускать последовательности стадий друг за другом;
- быстро сравнивать много запусков;
- иметь единый внешний UI для мониторинга;
- не терять аналитику, если внешний сервис недоступен.

Эта система проектируется под задачу улучшения reasoning и результатов на бенчмарках в рамках соревнования, а не как общий фреймворк для полного цикла обучения LLM с нуля.

## Scope

Входит:

- prompt и decoding-абляции;
- SFT через adapter-based fine-tuning;
- RL поверх лучшего SFT-чекпоинта;
- модульные reward packs;
- модульные eval packs;
- последовательные рецепты запусков;
- централизованный трекинг экспериментов.

Пока не входит:

- pretraining;
- full finetune всех параметров модели;
- универсальный DAG-оркестратор;
- отдельный сложный сервисный backend;
- избыточная платформенность без пользы для соревнования.

## Основные принципы

1. Один запуск должен полностью описываться набором конфигов.
2. Один и тот же checkpoint должен быть легко переиспользуем между SFT, RL и eval.
3. Любой stage должен иметь стандартные входы и выходы.
4. Все важные метрики и артефакты должны писаться в `W&B` и дублироваться локально.
5. Система должна быть удобной для серийных абляций, а не только для одиночного обучения.

## Целевые слои платформы

Платформа делится на несколько независимых слоев:

- `recipe layer`
- `config layer`
- `stage layer`
- `tracking layer`
- `local analytics layer`

`recipe layer` отвечает за порядок выполнения стадий и передачу артефактов.

`config layer` отвечает за сборку эксперимента из модульных конфигов.

`stage layer` отвечает за единый контракт исполнения и результатов.

`tracking layer` отвечает за внешний мониторинг и сравнение запусков через `W&B`.

`local analytics layer` отвечает за локальные таблицы и запросы через `DuckDB/Parquet`.

## Ближайшая цель

Минимальный рабочий контур должен покрыть:

1. `baseline_eval`
2. `sft`
3. `final_eval`
4. логирование в `W&B`
5. запись локальных таблиц в `Parquet`
6. локальную агрегацию через `DuckDB`
